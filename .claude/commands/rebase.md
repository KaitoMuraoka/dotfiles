# /rebase

---
description: 指定ブランチ（デフォルト: main）に対してgit rebaseを実行し、コンフリクトが発生した場合は内容を確認して修正する。
allowed-tools: Bash, Read, Edit, Glob, Grep
---

## 手順

1. `git status` で現在の状態を確認する
2. リベース対象ブランチを特定する（引数があればそれを使用、なければ `main` を使用）
3. `git rebase <ブランチ名>` を実行する
4. コンフリクトが発生した場合:
   - `git status` でコンフリクトしているファイルを確認する
   - 各ファイルを読み込み、`<<<<<<<`、`=======`、`>>>>>>>` マーカーを確認して内容を理解する
   - 両方の変更の意図を考慮した上でコンフリクトを解消する
   - 解消後、`git add <ファイル>` でステージングする
   - `git rebase --continue` を実行する
   - コンフリクトがなくなるまで繰り返す
5. リベース完了後、`git log --oneline -5` で結果を確認して報告する

## 注意事項
- コンフリクト解消時は、どちらの変更を採用したかを説明すること
- 判断が難しいコンフリクトはユーザーに確認すること
