# Claude Code をターミナルで開く

## なぜ作るか（Why）
Emacs 上で Claude Code を起動すると、TUI の表示が崩れて見づらく、動作も重い。
ターミナルアプリはインタラクティブな TUI に最適化されているため、Claude Code の起動先をターミナルに移すことで、快適な操作体験を実現する。

## 何を作るか（What）
Emacs から Claude Code CLI を起動する際に、Emacs 内ではなく外部ターミナルアプリで起動する Emacs Lisp 関数・コマンド。

具体的な動作：
1. 現在開いているバッファのディレクトリ、またはユーザーが指定したディレクトリを起動パスとして使用する
2. 設定したターミナルアプリ（未設定時はシステムデフォルト）を開く
3. そのターミナルで `claude` コマンドを実行する
4. ターミナルアプリにフォーカスを移す

## どう作るか（How）
- macOS の `open -a <TerminalApp>` または `osascript` を使ってターミナルを起動・フォーカス
- ターミナルアプリはカスタマイズ変数 `my/claude-terminal-app` で指定可能にする（デフォルト: `"Terminal"`）
- 対応ターミナル候補: Terminal.app, iTerm2, Alacritty, WezTerm など
- 起動ディレクトリは `default-directory`（現在バッファのディレクトリ）を使用し、`C-u` プレフィックスで任意ディレクトリを指定可能にする

## 決定事項（Decision Log）
- `vterm` や `eat` などの Emacs 内ターミナルは使わない。TUI の表示崩れと重さが根本問題のため、外部ターミナルへの委譲が正解。
- `osascript` を採用する理由: macOS 環境であり、ターミナルアプリへのフォーカス移動も `osascript` で確実に制御できるため。
